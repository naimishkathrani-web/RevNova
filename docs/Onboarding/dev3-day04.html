Day 4: Docker for Frontend

Developer 3 - DevOps & QA | Duration: ~6-8 hours

1 Understand Frontend Containerization

What you need to do: Learn how React apps run in Docker

Frontend in Docker:

  • Build step: Vite compiles React to static HTML/CSS/JS
  • Serve step: nginx web server serves the built files
  • nginx: Lightweight, fast web server (perfect for static files)
üí° Why nginx? React dev server (npm run dev) is NOT for production. nginx is production-ready and super fast!

2 Create Frontend Dockerfile

What you need to do: Write multi-stage Dockerfile for React

Create file: frontend/Dockerfile

# Stage 1: Build FROM node:20-alpine AS builder WORKDIR /app # Copy package files COPY package*.json ./ # Install dependencies RUN npm ci # Copy source code COPY . . # Build the app RUN npm run build # Stage 2: Production with nginx FROM nginx:alpine # Copy built files to nginx html directory COPY --from=builder /app/dist /usr/share/nginx/html # Copy custom nginx config COPY nginx.conf /etc/nginx/conf.d/default.conf # Expose port EXPOSE 80 # Start nginx CMD ["nginx", "-g", "daemon off;"]

3 Create nginx Configuration

What you need to do: Configure nginx for React Router

Create file: frontend/nginx.conf

server { listen 80; server_name localhost; root /usr/share/nginx/html; index index.html; # Enable gzip compression gzip on; gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript; # React Router support - serve index.html for all routes location / { try_files $uri $uri/ /index.html; } # Cache static assets location ~* \.(?:css|js|jpg|jpeg|gif|png|ico|svg|woff|woff2|ttf|eot)$ { expires 1y; add_header Cache-Control "public, immutable"; } }
üí° Why try_files? React Router uses client-side routing. nginx needs to always serve index.html so React can handle the routing!

4 Create Frontend .dockerignore

What you need to do: Exclude unnecessary files

Create file: frontend/.dockerignore

node_modules dist .env .git .gitignore README.md .vscode .idea coverage *.test.ts *.test.tsx *.spec.ts *.spec.tsx __tests__

5 Build Frontend Image

What you need to do: Create the Docker image

Build the image:

cd frontend docker build -t revnova-frontend:latest .

This will take 2-3 minutes. You'll see:

=> [builder 1/6] FROM node:20-alpine => [builder 2/6] WORKDIR /app ... => [stage-1 3/4] COPY --from=builder /app/dist /usr/share/nginx/html => => naming to docker.io/library/revnova-frontend:latest

Verify the image:

docker images | grep revnova

You should see both revnova-backend and revnova-frontend!

6 Run Frontend Container

What you need to do: Start and test the frontend

Run the container:

docker run -d -p 8080:80 --name revnova-frontend-container revnova-frontend:latest

Check if it's running:

docker ps

Open browser and go to: http://localhost:8080/

You should see the React app!

Test React Router:

  • Navigate to http://localhost:8080/wizard/connect
  • Refresh the page (F5)
  • Page should still load (not 404!) thanks to nginx config

Stop and remove:

docker stop revnova-frontend-container docker rm revnova-frontend-container
