Day 2: Database Tables (STG1)

Developer 1 - Backend & Database | Duration: ~6-8 hours

1 Understand the Database Strategy

What you need to do: Learn why we have 3 types of tables

RevNova uses 3 table types:

  1. STG1 Tables - Stores raw data exactly as it comes from Salesforce (what you'll build today)
  2. STG2 Tables - Stores normalized/cleaned data (Day 3)
  3. EAV Tables - Stores custom fields that users add (Day 4)
üí° Why 3 tables? STG1 keeps original data safe. STG2 has cleaned data for migration. EAV handles custom fields dynamically.

2 Install PostgreSQL

What you need to do: Install the database on your computer

  1. Go to https://www.postgresql.org/download/windows/
  2. Download PostgreSQL 15 or 16
  3. Run the installer
  4. Set password as: postgres (write this down!)
  5. Keep default port: 5432
  6. Click "Next" through remaining steps

Check if it installed:

psql --version

You should see:

psql (PostgreSQL) 15.x

3 Create RevNova Database

What you need to do: Create a new database for our project

Open terminal in VS Code and type:

psql -U postgres

When it asks for password, type: postgres

Now create the database:

CREATE DATABASE revnova_dev;
\l

You should see revnova_dev in the list. Type \q to exit.

4 Create Migration File for STG1 Table

What you need to do: Create a file that will build the STG1 table

In VS Code, create this file:

Location: backend/src/database/migrations/001_create_stg1_table.sql

Copy and paste this code:

-- STG1: Raw data staging table CREATE TABLE IF NOT EXISTS stg1_raw_data ( id SERIAL PRIMARY KEY, migration_id VARCHAR(50) NOT NULL, source_system VARCHAR(50) NOT NULL DEFAULT 'SFDC', object_name VARCHAR(100) NOT NULL, record_id VARCHAR(50) NOT NULL, raw_data JSONB NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Indexes for faster queries CREATE INDEX idx_stg1_migration ON stg1_raw_data(migration_id); CREATE INDEX idx_stg1_object ON stg1_raw_data(object_name); CREATE INDEX idx_stg1_record ON stg1_raw_data(record_id);
üí° What does this do? Creates a table that stores raw Salesforce data in JSON format. The indexes make searching faster.

5 Run the Migration

What you need to do: Execute the SQL file to create the table

Run this command in terminal:

psql -U postgres -d revnova_dev -f backend/src/database/migrations/001_create_stg1_table.sql

Check if it worked:

psql -U postgres -d revnova_dev -c "\dt"

You should see stg1_raw_data table in the list!

6 Test the Table

What you need to do: Insert sample data to verify table works

Connect to database:

psql -U postgres -d revnova_dev

Insert test data:

INSERT INTO stg1_raw_data (migration_id, source_system, object_name, record_id, raw_data) VALUES ('TEST001', 'SFDC', 'Quote', 'Q001', '{"name": "Test Quote", "amount": 5000}');

Query the data:

SELECT * FROM stg1_raw_data;

You should see your test record! Type \q to exit.
