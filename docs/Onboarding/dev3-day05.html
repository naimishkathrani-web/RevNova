Day 5: Code Coverage & Week 1 PR

Developer 3 - DevOps & QA | Duration: ~6-8 hours

1 Configure Jest Coverage for Backend

What you need to do: Setup code coverage reporting

Update backend/jest.config.js to add coverage settings:

module.exports = { preset: 'ts-jest', testEnvironment: 'node', roots: ['/src'], testMatch: ['**/__tests__/**/*.ts', '**/?(*.)+(spec|test).ts'], moduleFileExtensions: ['ts', 'js', 'json'], collectCoverageFrom: [ 'src/**/*.ts', '!src/**/*.d.ts', '!src/**/*.test.ts', '!src/**/*.spec.ts', ], coverageThreshold: { global: { branches: 80, functions: 80, lines: 80, statements: 80 } }, coverageReporters: ['text', 'lcov', 'html'], };
üí° What's coverageThreshold? Build fails if code coverage is below 80%. This ensures good test quality!

2 Run Coverage Locally

What you need to do: Generate coverage reports

Run tests with coverage:

cd backend npm run test:coverage

You should see coverage report:

----------|---------|----------|---------|---------| File | % Stmts | % Branch | % Funcs | % Lines | ----------|---------|----------|---------|---------| All files | 100 | 100 | 100 | 100 | ----------|---------|----------|---------|---------|

Open the HTML report:

start backend\coverage\index.html

You'll see a beautiful interactive coverage report!

3 Update CI Workflow with Coverage

What you need to do: Add coverage to GitHub Actions

Update .github/workflows/ci.yml to add coverage job:

- name: Test backend with coverage working-directory: ./backend run: npm run test:coverage - name: Upload backend coverage uses: codecov/codecov-action@v3 with: files: ./backend/coverage/lcov.info flags: backend name: backend-coverage - name: Test frontend with coverage working-directory: ./frontend run: npm run test:coverage - name: Upload frontend coverage uses: codecov/codecov-action@v3 with: files: ./frontend/coverage/lcov.info flags: frontend name: frontend-coverage

4 Add Coverage Badge to README

What you need to do: Show coverage status in README

Add badges to top of README.md:

# RevNova Migration Tool ![CI Pipeline](https://github.com/naimishkathrani-web/RevNova/actions/workflows/ci.yml/badge.svg) ![Coverage](https://codecov.io/gh/naimishkathrani-web/RevNova/branch/main/graph/badge.svg) Automated data migration tool for Salesforce CPQ to CPQ Billing.
üí° Badges: Visual indicators of project health shown on GitHub homepage!

5 Create docker-compose.yml

What you need to do: Run both containers together

Create file in root: docker-compose.yml

version: '3.8' services: backend: build: ./backend container_name: revnova-backend ports: - "3000:3000" environment: - NODE_ENV=production - PORT=3000 networks: - revnova-network frontend: build: ./frontend container_name: revnova-frontend ports: - "8080:80" depends_on: - backend networks: - revnova-network networks: revnova-network: driver: bridge

Test docker-compose:

docker-compose up -d docker-compose ps docker-compose logs docker-compose down

6 Commit Week 1 Work

What you need to do: Save all Week 1 progress

Check what changed:

git status

Add and commit everything:

git add . git commit -m "feat: Week 1 complete - CI/CD pipeline, Docker, code coverage - GitHub Actions CI pipeline with lint and test - ESLint configured for backend and frontend - Jest tests with 80% coverage threshold - Multi-stage Dockerfiles for backend and frontend - nginx config for React Router - docker-compose for local development - Coverage reporting with Codecov" git push origin main
