Day 1: Workstation Setup

Developer 1: Backend & Database | Duration: ~6-8 hours

üìã Objective

Set up your development workstation with all required tools and services for backend development. By the end of this day, you'll have Node.js, PostgreSQL, Redis, and the RevNova backend running locally.

What You'll Accomplish:

  • Install Node.js LTS (v18+) and npm
  • Install and configure PostgreSQL 16
  • Install and configure Redis 7
  • Clone the repository and install dependencies
  • Set up environment variables
  • Run database migrations
  • Start the backend server successfully

üíª Step 1: Install Node.js and npm

üí¨ GitHub Copilot Prompt:

# Set up Node.js LTS development environment
# Install PostgreSQL 15+ and Redis
# Verify installations with version commands

Download and install Node.js LTS from nodejs.org

Verify installation:

node --version
# Should show v18.x.x or higher

npm --version
# Should show 9.x.x or higher

‚úÖ Checkpoint: Node.js and npm are installed and accessible from command line

üêò Step 2: Install PostgreSQL

üí¨ GitHub Copilot Prompt:

# Clone RevNova repository from GitHub
# Run npm install to install all dependencies
# Create .env file from .env.example template

Download PostgreSQL 16 from postgresql.org

During installation:

  • Set superuser password (remember this!)
  • Use default port 5432
  • Include pgAdmin 4 and Stack Builder

Verify PostgreSQL is running:

# Windows
netstat -ano | findstr :5432

# macOS/Linux
lsof -i :5432

Create RevNova database:

psql -U postgres
CREATE DATABASE revnova_dev;
\q

‚úÖ Checkpoint: PostgreSQL is running on port 5432, revnova_dev database created

‚ö° Step 3: Install Redis

üí¨ GitHub Copilot Prompt:

# Configure PostgreSQL connection in .env file
# Set DATABASE_URL with host, port, database, username, password
# Add Redis connection URL for job queuing

Windows: Download from GitHub or use WSL

macOS: brew install redis

Linux: sudo apt-get install redis-server

Start Redis:

# Windows (if installed directly)
redis-server

# macOS
brew services start redis

# Linux
sudo systemctl start redis

Verify Redis is running:

# Test connection
redis-cli ping
# Should return: PONG

‚úÖ Checkpoint: Redis is running on port 6379

üìÅ Step 4: Clone Repository and Install Dependencies

üí¨ GitHub Copilot Prompt:

# Run database migrations using npm run migrate
# Verify all tables created successfully
# Check migrations table for applied migrations

Clone the RevNova repository:

git clone https://github.com/naimishkathrani-web/RevNova.git
cd RevNova/backend

Install backend dependencies:

npm install

This will install all required packages including:

  • Express - Web framework
  • TypeScript - Type safety
  • pg (node-postgres) - PostgreSQL client
  • redis - Redis client
  • jsforce - Salesforce API client
  • And many more...

‚úÖ Checkpoint: All npm packages installed successfully, node_modules folder created

‚öôÔ∏è Step 5: Configure Environment Variables

üí¨ GitHub Copilot Prompt:

# Start development server with npm run dev
# Verify server runs on port 3000
# Test health endpoint at http://localhost:3000/api/v1/health

Create a .env file in the backend directory:

File: backend/.env

NODE_ENV=development
PORT=3000

# PostgreSQL
DB_HOST=localhost
DB_PORT=5432
DB_NAME=revnova_dev
DB_USER=postgres
DB_PASSWORD=your_postgres_password

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# Salesforce (will configure later)
SF_CLIENT_ID=
SF_CLIENT_SECRET=
SF_CALLBACK_URL=http://localhost:3000/api/v1/auth/callback

# JWT Secret
JWT_SECRET=your_random_secret_key_here

‚ö†Ô∏è Important: Replace your_postgres_password with your actual PostgreSQL password

‚úÖ Checkpoint: .env file created with all required variables

ÔøΩÔ∏è Step 6: Run Database Migrations

Run the database migrations to create all tables:

npm run migrate

This will create the following tables:

  • projects - Migration projects
  • connections - Salesforce connections
  • source_objects - SFDC source objects
  • source_fields - SFDC source fields
  • target_objects - Target custom objects
  • target_fields - Target custom fields
  • field_mappings - Field mapping definitions

Verify tables were created:

psql -U postgres -d revnova_dev
\dt
# Should list all tables
\q

‚úÖ Checkpoint: All database tables created successfully

üêõ Troubleshooting

Issue: PostgreSQL connection error

  • Verify PostgreSQL is running: netstat -ano | findstr :5432
  • Check credentials in .env file
  • Ensure revnova_dev database exists

Issue: Redis connection error

  • Verify Redis is running: redis-cli ping
  • Check port 6379 is not blocked
  • On Windows, may need to install via WSL

Issue: npm install fails

  • Clear npm cache: npm cache clean --force
  • Delete node_modules and package-lock.json
  • Run npm install again
