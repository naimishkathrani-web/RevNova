Day 3: Database Tables (STG2)

Developer 1 - Backend & Database | Duration: ~6-8 hours

1 Understand STG2 Tables

What you need to do: Learn what STG2 tables are for

STG2 tables store normalized data:

  • STG1 = Raw JSON data from Salesforce
  • STG2 = Clean, structured data ready for migration
  • We'll create 3 tables: quotes, line_items, products
üí° Why normalize? JSON is flexible but slow. Relational tables are faster and easier to query!

2 Create Quotes Table Migration

What you need to do: Create the quotes table

Create file: backend/src/database/migrations/002_create_stg2_quotes.sql

-- STG2: Normalized quotes table CREATE TABLE IF NOT EXISTS stg2_quotes ( id SERIAL PRIMARY KEY, migration_id VARCHAR(50) NOT NULL, quote_id VARCHAR(50) NOT NULL UNIQUE, quote_number VARCHAR(50), account_name VARCHAR(255), contact_name VARCHAR(255), total_amount DECIMAL(15,2), status VARCHAR(50), created_date DATE, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (migration_id) REFERENCES migrations(id) ON DELETE CASCADE ); CREATE INDEX idx_stg2_quotes_migration ON stg2_quotes(migration_id); CREATE INDEX idx_stg2_quotes_id ON stg2_quotes(quote_id);

Run the migration:

psql -U postgres -d revnova_dev -f backend/src/database/migrations/002_create_stg2_quotes.sql

3 Create Line Items Table

What you need to do: Create the line_items table

Create file: backend/src/database/migrations/003_create_stg2_line_items.sql

-- STG2: Line items table CREATE TABLE IF NOT EXISTS stg2_line_items ( id SERIAL PRIMARY KEY, migration_id VARCHAR(50) NOT NULL, line_item_id VARCHAR(50) NOT NULL UNIQUE, quote_id VARCHAR(50) NOT NULL, product_id VARCHAR(50), quantity INTEGER, unit_price DECIMAL(15,2), line_total DECIMAL(15,2), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (quote_id) REFERENCES stg2_quotes(quote_id) ON DELETE CASCADE ); CREATE INDEX idx_stg2_line_items_quote ON stg2_line_items(quote_id); CREATE INDEX idx_stg2_line_items_product ON stg2_line_items(product_id);

Run the migration:

psql -U postgres -d revnova_dev -f backend/src/database/migrations/003_create_stg2_line_items.sql

4 Create Products Table

What you need to do: Create the products table

Create file: backend/src/database/migrations/004_create_stg2_products.sql

-- STG2: Products table CREATE TABLE IF NOT EXISTS stg2_products ( id SERIAL PRIMARY KEY, migration_id VARCHAR(50) NOT NULL, product_id VARCHAR(50) NOT NULL UNIQUE, product_code VARCHAR(50), product_name VARCHAR(255), description TEXT, list_price DECIMAL(15,2), is_active BOOLEAN DEFAULT true, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); CREATE INDEX idx_stg2_products_migration ON stg2_products(migration_id); CREATE INDEX idx_stg2_products_id ON stg2_products(product_id);

Run the migration:

psql -U postgres -d revnova_dev -f backend/src/database/migrations/004_create_stg2_products.sql

5 Create Migrations Tracking Table

What you need to do: Create table to track migrations

Create file: backend/src/database/migrations/000_create_migrations.sql

-- Migrations tracking table CREATE TABLE IF NOT EXISTS migrations ( id VARCHAR(50) PRIMARY KEY, name VARCHAR(255) NOT NULL, source_system VARCHAR(50) DEFAULT 'SFDC', target_system VARCHAR(50) DEFAULT 'CPQB', status VARCHAR(50) DEFAULT 'pending', created_by VARCHAR(100), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );

Run the migration:

psql -U postgres -d revnova_dev -f backend/src/database/migrations/000_create_migrations.sql

6 Test All Tables

What you need to do: Verify all tables exist

Connect to database:

psql -U postgres -d revnova_dev

List all tables:

\dt

You should see:

  • migrations
  • stg1_raw_data
  • stg2_quotes
  • stg2_line_items
  • stg2_products

Type \q to exit.
