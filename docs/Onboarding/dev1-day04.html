1 Understand EAV Pattern

What you need to do: Learn what EAV is and why we need it

EAV = Entity-Attribute-Value Pattern

  • Entity: The record (like a Quote)
  • Attribute: The custom field name (like "Warranty_Period__c")
  • Value: The field's value (like "12 months")
üí° Why EAV? Salesforce lets users create custom fields. We can't predict what fields they'll add, so EAV stores them dynamically!

2 Create Custom Fields Table

What you need to do: Create table to store custom field definitions

Create file: backend/src/database/migrations/005_create_eav_custom_fields.sql

-- EAV: Custom field definitions CREATE TABLE IF NOT EXISTS eav_custom_fields ( id SERIAL PRIMARY KEY, migration_id VARCHAR(50) NOT NULL, object_name VARCHAR(100) NOT NULL, field_name VARCHAR(100) NOT NULL, field_label VARCHAR(255), field_type VARCHAR(50), is_required BOOLEAN DEFAULT false, default_value TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, UNIQUE(migration_id, object_name, field_name) ); CREATE INDEX idx_eav_fields_migration ON eav_custom_fields(migration_id); CREATE INDEX idx_eav_fields_object ON eav_custom_fields(object_name);

Run the migration:

psql -U postgres -d revnova_dev -f backend/src/database/migrations/005_create_eav_custom_fields.sql

3 Create Custom Values Table

What you need to do: Create table to store actual custom field values

Create file: backend/src/database/migrations/006_create_eav_custom_values.sql

-- EAV: Custom field values CREATE TABLE IF NOT EXISTS eav_custom_values ( id SERIAL PRIMARY KEY, migration_id VARCHAR(50) NOT NULL, object_name VARCHAR(100) NOT NULL, record_id VARCHAR(50) NOT NULL, field_id INTEGER NOT NULL, value_text TEXT, value_number DECIMAL(18,6), value_date DATE, value_boolean BOOLEAN, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (field_id) REFERENCES eav_custom_fields(id) ON DELETE CASCADE ); CREATE INDEX idx_eav_values_migration ON eav_custom_values(migration_id); CREATE INDEX idx_eav_values_record ON eav_custom_values(record_id); CREATE INDEX idx_eav_values_field ON eav_custom_values(field_id);

Run the migration:

psql -U postgres -d revnova_dev -f backend/src/database/migrations/006_create_eav_custom_values.sql

4 Test EAV Pattern

What you need to do: Insert test data to verify EAV works

Connect to database:

psql -U postgres -d revnova_dev

Step 1: Add a migration record:

INSERT INTO migrations (id, name, source_system, target_system, created_by) VALUES ('MIG001', 'Test Migration', 'SFDC', 'CPQB', 'admin');

Step 2: Add a custom field definition:

INSERT INTO eav_custom_fields (migration_id, object_name, field_name, field_label, field_type) VALUES ('MIG001', 'Quote', 'Warranty_Period__c', 'Warranty Period', 'text');

Step 3: Add a custom field value:

INSERT INTO eav_custom_values (migration_id, object_name, record_id, field_id, value_text) VALUES ('MIG001', 'Quote', 'Q001', 1, '12 months');

5 Query EAV Data

What you need to do: Test querying custom fields with a JOIN

Run this query to see custom fields with their values:

SELECT v.record_id, f.field_label, f.field_type, v.value_text FROM eav_custom_values v JOIN eav_custom_fields f ON v.field_id = f.id WHERE v.record_id = 'Q001';

You should see:

record_id | field_label | field_type | value_text ----------+------------------+------------+------------ Q001 | Warranty Period | text | 12 months

Type \q to exit.

6 Verify All Database Tables

What you need to do: Check that all 8 tables exist

Connect to database:

psql -U postgres -d revnova_dev -c "\dt"

You should see all these tables:

  • stg1_raw_data (raw JSON data)

  • stg2_line_items (quote line items)
  • stg2_products (product catalog)
  • eav_custom_fields (custom field definitions)
  • eav_custom_values (custom field values)
