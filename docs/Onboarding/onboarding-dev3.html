Developer 3 — DevOps, Queue & QA

Focus: Redis/Bull infra, worker CI, lint/test gates, deployment workflows, and Dev→Test→Prod pipeline.

Software & Tools (Windows) — Download & Install

  • Git: https://git-scm.com/download/win
  • Docker Desktop: https://www.docker.com/products/docker-desktop/
  • VS Code: https://code.visualstudio.com/ (Copilot and Actions extensions helpful)
  • GitHub CLI (gh): https://github.com/cli/cli
  • Optional: kubectl or Terraform if infra-as-code is introduced later

Clone RevNova-Dev and add test and prod remotes pointing to RevNova-Test and RevNova.

Queues & Workers

  • Provision Redis (local via Docker for dev).
  • Create Bull queues for extract/transform/load/validate.
  • Implement retry policies, backoff, and dead-letter behavior.
# Example Redis (dev only)
# docker run -p 6379:6379 redis:7-alpine

CI/CD

  • GitHub Actions: lint, typecheck, unit tests on PRs.
  • Optional: build and publish Docker images on main merges.
  • Test summaries and artifacts (coverage reports).

Environment Promotion (Dev → Test → Prod)

  1. Dev: PRs to Dev main require checks (lint/test). Tag preview releases if needed.
  2. Test: Create a release branch and push to test remote. Open PR in RevNova-Test with Copilot-generated description. Run integration tests.
  3. Prod: After QA sign-off, open PR to RevNova. Use protected branches and require approvals. Tag semantic version on merge.

Automate promotions with reusable GitHub Actions workflows and required environment approvals.

Quality Gates

  • Build: Node backend builds without errors.
  • Lint/Typecheck: ESLint + TypeScript pass.
  • Tests: core API handlers and worker logic green.

Deliverables

  • CI pipelines defined.
  • Reusable worker image or process manager config.
  • Documentation updates for local/CI runs.
