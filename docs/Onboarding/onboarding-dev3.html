Developer 3 — DevOps, Queue & QA

Focus: Redis/Bull infra, worker CI, lint/test gates, and basic deployment workflows.

Setup

  • Install Docker Desktop and Git.
  • Access to CI provider (GitHub Actions) and cloud secrets store.
  • Create branch: chore/ci-infra.

Queues & Workers

  • Provision Redis (local via Docker for dev).
  • Create Bull queues for extract/transform/load/validate.
  • Implement retry policies, backoff, and dead-letter behavior.
# Example Redis (dev only)
# docker run -p 6379:6379 redis:7-alpine

CI/CD

  • GitHub Actions: lint, typecheck, unit tests on PRs.
  • Optional: build and publish Docker images on main merges.
  • Test summaries and artifacts (coverage reports).

Quality Gates

  • Build: Node backend builds without errors.
  • Lint/Typecheck: ESLint + TypeScript pass.
  • Tests: core API handlers and worker logic green.

Deliverables

  • CI pipelines defined.
  • Reusable worker image or process manager config.
  • Documentation updates for local/CI runs.
