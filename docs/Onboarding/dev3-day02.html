Day 2: CI Pipeline (Lint & Test)

Developer 3 - DevOps & QA | Duration: ~6-8 hours

1 Understand CI/CD Concepts

What you need to do: Learn what CI/CD means

CI/CD Definitions:

  • CI (Continuous Integration): Automatically test code when developers push changes
  • CD (Continuous Delivery): Automatically deploy code when tests pass
  • Pipeline: A series of automated steps (lint ‚Üí test ‚Üí build ‚Üí deploy)
üí° Why CI/CD? Catches bugs early, ensures code quality, and automates repetitive tasks!

2 Create CI Workflow File

What you need to do: Create GitHub Actions workflow for CI

Create file: .github/workflows/ci.yml

name: CI Pipeline on: push: branches: [main, develop] pull_request: branches: [main, develop] jobs: lint-and-test: runs-on: ubuntu-latest strategy: matrix: node-version: [20.x] steps: - name: Checkout code uses: actions/checkout@v4 - name: Setup Node.js uses: actions/setup-node@v4 with: node-version: ${{ matrix.node-version }} cache: 'npm' - name: Install backend dependencies working-directory: ./backend run: npm ci - name: Install frontend dependencies working-directory: ./frontend run: npm ci - name: Lint backend working-directory: ./backend run: npm run lint - name: Lint frontend working-directory: ./frontend run: npm run lint - name: Test backend working-directory: ./backend run: npm test - name: Test frontend working-directory: ./frontend run: npm test

3 Add ESLint to Backend

What you need to do: Configure code linting for backend

Navigate to backend and install ESLint:

cd backend npm install --save-dev eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin npx eslint --init

When prompted, choose:

  • ‚úÖ To check syntax, find problems, and enforce code style
  • ‚úÖ JavaScript modules (import/export)
  • ‚úÖ None of these (we're not using React/Vue)
  • ‚úÖ Yes (TypeScript)
  • ‚úÖ Node

Add lint script to backend/package.json:

"scripts": { "lint": "eslint . --ext .ts", "lint:fix": "eslint . --ext .ts --fix" }

4 Add ESLint to Frontend

What you need to do: Configure linting for React frontend

Vite already includes ESLint. Just add lint scripts to frontend/package.json:

"scripts": { "lint": "eslint . --ext ts,tsx --report-unused-disable-directives --max-warnings 0", "lint:fix": "eslint . --ext ts,tsx --fix" }

Test linting locally:

cd frontend npm run lint

5 Setup Jest for Backend Tests

What you need to do: Configure unit testing

Install Jest for backend:

cd backend npm install --save-dev jest @types/jest ts-jest

Create backend/jest.config.js:

module.exports = { preset: 'ts-jest', testEnvironment: 'node', roots: ['/src'], testMatch: ['**/__tests__/**/*.ts', '**/?(*.)+(spec|test).ts'], moduleFileExtensions: ['ts', 'js', 'json'], collectCoverageFrom: [ 'src/**/*.ts', '!src/**/*.d.ts', ], };

Add test script to backend/package.json:

"scripts": { "test": "jest", "test:watch": "jest --watch", "test:coverage": "jest --coverage" }

6 Write a Simple Test

What you need to do: Create a test file to verify setup works

Create backend/src/__tests__/sample.test.ts:

describe('Sample Test', () => { it('should pass basic test', () => { expect(1 + 1).toBe(2); }); it('should verify environment', () => { expect(process.env.NODE_ENV).toBeDefined(); }); });

Run the test:

npm test

You should see: ‚úì should pass basic test

7 Push and Verify CI

What you need to do: Commit and watch GitHub Actions run

Commit your changes:

git add . git commit -m "ci: Add GitHub Actions CI pipeline with lint and test" git push origin main

Go to GitHub and check your workflow:

  1. Open https://github.com/naimishkathrani-web/RevNova
  2. Click "Actions" tab
  3. You should see "CI Pipeline" workflow running
  4. Click on the workflow to see live logs
  5. Wait for green checkmark ‚úÖ
üí° If it fails: Click on the failed job to see error logs. Common issues: missing package.json scripts, syntax errors in YAML.
