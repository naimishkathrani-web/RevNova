Developer 1 — Backend & Database

Focus: Phase 1 APIs, DB schema (Vanilla + EAV), validation and batch execution services, plus Dev→Test promotion.

Software & Tools (Windows) — Download & Install

  • VS Code: https://code.visualstudio.com/ (User Installer). Sign in to GitHub in VS Code.
  • Git: https://git-scm.com/download/win (enable Git from command line).
  • Node.js 20.x: https://nodejs.org/en/download (verify: node -v).
  • Python 3.11+: https://www.python.org/downloads/windows/ (add to PATH).
  • PostgreSQL client: pgAdmin 4 https://www.pgadmin.org/download/ or Azure Data Studio + PostgreSQL extension.
  • Docker Desktop (optional for local DB/Redis): https://www.docker.com/products/docker-desktop/
  • GitHub CLI (gh): https://github.com/cli/cli

Optional commands are shown for reference and may be run in PowerShell when needed.

Repository Access & VS Code Setup

  • Access RevNova-Dev (not Production). Clone into C:\\Dev\\RevNovaRepository.
  • Open folder in VS Code; set terminal to Windows PowerShell.
  • Install extensions: GitHub Copilot, GitHub Copilot Chat, ESLint, Prettier, GitLens, REST Client, Docker, YAML, Python, Pylance.
  • Enable Copilot for this workspace; use it to scaffold API handlers and tests.
  • Add test remote pointing to RevNova-Test for later promotion.
  • Create branch: feature/api-mvp.

Database

  • Connect to the shared PostgreSQL using read/write credentials if assigned.
  • Implement schemas for STG1 (raw) and STG2 (normalized) plus EAV tables for custom fields.
  • Create seed scripts for basic objects (Product2, Pricebook2, Quote, etc.).
-- Example: EAV table sketch
CREATE TABLE IF NOT EXISTS stg_eav (
  id BIGSERIAL PRIMARY KEY,
  object_name TEXT NOT NULL,
  record_id TEXT NOT NULL,
  field_name TEXT NOT NULL,
  field_type TEXT NOT NULL,
  field_value TEXT,
  stage TEXT NOT NULL CHECK (stage IN ('STG1','STG2','TARGET'))
);

APIs (Phase 1)

  • Connection: POST /api/v1/connections/test, POST /api/v1/connections
  • Schema: GET /api/v1/schema/source, GET /api/v1/schema/target
  • Mapping: POST /api/v1/mappings, GET /api/v1/mappings/:object
  • Transform: POST /api/v1/transform/preview
  • Validate: POST /api/v1/validate/run
  • Execute: POST /api/v1/migrate/run, GET /api/v1/migrate/status
  • Test: POST /api/v1/test/sample, GET /api/v1/test/compare

Queue & Workers

  • Use Bull (Redis) for batch jobs. Define queues for extract, transform, load, validate.
  • Implement exponential backoff and idempotent operations.

Unit Testing & Local Runs

  • Testing stack: Jest + ts-jest for unit tests; Supertest for API endpoints.
  • Write tests for connection tests, mapping save/lookup, and validation kickoffs.
  • Ensure lint and typecheck pass before commits. Use Copilot to propose tests and refactors.

Deliverables

  • DB schema DDL with migrations.
  • API stubs with request/response validation.
  • Worker processes with configurable concurrency.
  • Basic unit tests for API handlers.

Promote Code to Test (with Copilot help)

  1. Merge your feature PR into Dev main.
  2. Update local; create a release branch (e.g., release/feature-x).
  3. Push branch to test remote (RevNova-Test).
  4. Open PR in RevNova-Test. Use Copilot Chat to draft PR description and checklist.
  5. After QA, follow team process to open PR to Production.
