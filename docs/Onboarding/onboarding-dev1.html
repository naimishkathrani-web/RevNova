Developer 1 — Backend & Database

Focus: Phase 1 APIs, DB schema (Vanilla + EAV), validation and batch execution services.

Setup

  • Install Node.js 20, Git, and a PostgreSQL client (psql/pgAdmin).
  • Clone the repo and open in VS Code.
  • Create branch: feature/api-mvp.

Database

  • Connect to the shared PostgreSQL using read/write credentials if assigned.
  • Implement schemas for STG1 (raw) and STG2 (normalized) plus EAV tables for custom fields.
  • Create seed scripts for basic objects (Product2, Pricebook2, Quote, etc.).
-- Example: EAV table sketch
CREATE TABLE IF NOT EXISTS stg_eav (
  id BIGSERIAL PRIMARY KEY,
  object_name TEXT NOT NULL,
  record_id TEXT NOT NULL,
  field_name TEXT NOT NULL,
  field_type TEXT NOT NULL,
  field_value TEXT,
  stage TEXT NOT NULL CHECK (stage IN ('STG1','STG2','TARGET'))
);

APIs (Phase 1)

  • Connection: POST /api/v1/connections/test, POST /api/v1/connections
  • Schema: GET /api/v1/schema/source, GET /api/v1/schema/target
  • Mapping: POST /api/v1/mappings, GET /api/v1/mappings/:object
  • Transform: POST /api/v1/transform/preview
  • Validate: POST /api/v1/validate/run
  • Execute: POST /api/v1/migrate/run, GET /api/v1/migrate/status
  • Test: POST /api/v1/test/sample, GET /api/v1/test/compare

Queue & Workers

  • Use Bull (Redis) for batch jobs. Define queues for extract, transform, load, validate.
  • Implement exponential backoff and idempotent operations.

Deliverables

  • DB schema DDL with migrations.
  • API stubs with request/response validation.
  • Worker processes with configurable concurrency.
  • Basic unit tests for API handlers.
