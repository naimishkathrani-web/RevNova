Day 3: Docker for Backend

Developer 3 - DevOps & QA | Duration: ~6-8 hours

1 Understand Docker Basics

What you need to do: Learn Docker fundamentals

Key Docker Concepts:

  • Image: A blueprint/template for containers (like a .exe installer)
  • Container: Running instance of an image (like a running program)
  • Dockerfile: Recipe to build an image (list of instructions)
  • Multi-stage build: Build in stages to keep final image small
üí° Why Docker? "Works on my machine" ‚Üí Works everywhere! Same environment for dev, test, and production.

2 Create Backend Dockerfile

What you need to do: Write Dockerfile for Node.js backend

Create file: backend/Dockerfile

# Stage 1: Build FROM node:20-alpine AS builder WORKDIR /app # Copy package files COPY package*.json ./ # Install dependencies RUN npm ci # Copy source code COPY . . # Build TypeScript RUN npm run build # Stage 2: Production FROM node:20-alpine WORKDIR /app # Copy package files COPY package*.json ./ # Install production dependencies only RUN npm ci --only=production # Copy built files from builder stage COPY --from=builder /app/dist ./dist # Expose port EXPOSE 3000 # Set environment ENV NODE_ENV=production # Run the app CMD ["node", "dist/index.js"]

3 Create .dockerignore File

What you need to do: Exclude unnecessary files from Docker image

Create file: backend/.dockerignore

node_modules npm-debug.log .env .git .gitignore README.md .vscode .idea coverage *.test.ts *.spec.ts __tests__
üí° Why .dockerignore? Smaller images = faster builds = faster deployments!

4 Build Docker Image

What you need to do: Create the backend Docker image

Make sure Docker Desktop is running (whale icon in system tray)

Build the image:

cd backend docker build -t revnova-backend:latest .

This will take 2-3 minutes. You'll see:

=> [builder 1/6] FROM node:20-alpine => [builder 2/6] WORKDIR /app => [builder 3/6] COPY package*.json ./ ... => => naming to docker.io/library/revnova-backend:latest

Verify the image was created:

docker images

You should see revnova-backend in the list!

5 Run Backend Container

What you need to do: Start a container from the image

Run the container:

docker run -d -p 3000:3000 --name revnova-backend-container revnova-backend:latest

What do these flags mean?

  • -d = detached (run in background)
  • -p 3000:3000 = map port 3000 (host:container)
  • --name = give container a friendly name

Check if it's running:

docker ps

You should see your container in the list!

6 Test the Containerized API

What you need to do: Verify the API works in Docker

Check container logs:

docker logs revnova-backend-container

You should see: üöÄ Server running on http://localhost:3000

Test the health endpoint:

curl http://localhost:3000/api/v1/health

Or open in browser: http://localhost:3000/api/v1/health

Stop and remove the container:

docker stop revnova-backend-container docker rm revnova-backend-container
