Developer 2 — Frontend (React)

Focus: 7-step migration wizard UI, mapping editor, transformation builder, validation and job console, plus Dev→Test promotion.

Software & Tools (Windows) — Download & Install

  • VS Code: https://code.visualstudio.com/ (User Installer). Sign in to GitHub in VS Code.
  • Git: https://git-scm.com/download/win (enable Git from command line).
  • Node.js 20.x: https://nodejs.org/en/download (verify: node -v). Package manager: npm or pnpm.
  • Python 3.11+: https://www.python.org/downloads/windows/ (add to PATH).
  • Browsers: Chrome + DevTools; optionally Edge.
  • GitHub CLI (gh): https://github.com/cli/cli

Repository Access & VS Code Setup

  • Access RevNova-Dev only. Clone into C:\\Dev\\RevNovaRepository.
  • Open folder in VS Code; set terminal to Windows PowerShell.
  • Install extensions: GitHub Copilot, GitHub Copilot Chat, ESLint, Prettier, GitLens, Tailwind CSS IntelliSense, React, REST Client.
  • Enable Copilot for this workspace; use it to scaffold components, hooks, tests.
  • Add test remote pointing to RevNova-Test for later promotion.
  • Create branch: feature/wizard-ui.

UI Scope

  • Scaffold React 18 + TypeScript + TailwindCSS.
  • Implement pages: Connect, Analyze, Mapping, Transform, Validate, Execute, Test.
  • Mapping UI: table with AI suggestions, confidence score, manual override, search/filter.
  • Transform builder: expression editor + test preview.
  • Validate: error table with severity filters and CSV export.
  • Execute: progress bar, batch/job list, pause/resume/retry controls.

API Contracts

  • Use the endpoints defined in Phase 1 Technical.
  • Create a small API client with typed responses and error handling.
  • Use React Query or SWR for data fetching and caching.

Testing & Local Runs

  • Use Vitest/Jest + React Testing Library for unit tests.
  • Test mapping table interactions, transform editor parsing, and step navigation.
  • Ensure ESLint + Prettier pass before commits. Use Copilot to propose tests and refactors.

Deliverables

  • Functional wizard navigation with step state persisted in URL/query.
  • Responsive UI with accessibility checks.
  • Unit tests for key components (mapping table, transform editor).

Promote Code to Test (with Copilot help)

  1. Merge your feature PR into Dev main.
  2. Create a release branch and push to the test remote.
  3. Open PR in RevNova-Test; use Copilot Chat to draft PR description.
  4. After QA, coordinate Production PR with the team.
