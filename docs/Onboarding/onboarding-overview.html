RevNova Onboarding

Everything you need to set up your machine, access the repository, connect to our shared PostgreSQL database, and start building.

Project Overview

  • Product: RevNova — AI-powered migration and product configuration platform.
  • Frontend: React 18 + TypeScript + TailwindCSS.
  • Backend: Node.js 20 + Express + TypeScript.
  • Database: PostgreSQL 15/16 (multi-tenant) with hybrid staging (Vanilla + EAV).
  • Queue: Bull (Redis 7).
  • AI: OpenAI GPT-4 for mapping and content generation.
  • Storage: AWS S3/Azure Blob for artifacts.

Requirements are organized per phase and per view. Start at Requirements Home.

1) Machine Prerequisites (Windows)

  • VS Code (latest) + Extensions: ESLint, Prettier, GitLens, Docker, YAML, REST Client, Thunder Client, Python, Pylance.
  • Node.js 20.x and npm 10.x
  • Git (with SSH configured)
  • Docker Desktop (optional for local services)
  • PostgreSQL client (psql) and a GUI (pgAdmin 4 or Azure Data Studio)
  • Redis (local optional) or use Docker image

2) Repository Access

  • Ensure your GitHub account has access to the RevNova repository.
  • Set up SSH keys in GitHub and your machine (ssh-keygen, add to GitHub).
  • Clone the repository to your dev folder (ex: C:\\Dev\\RevNovaRepository).
  • Use a feature-branch workflow: feature/<area>-<short-desc>.

3) VS Code Workspace

  • Open the folder RevNovaRepository in VS Code.
  • Enable recommended extensions: run scripts/install-extensions.ps1 if provided.
  • Set default shell: Windows PowerShell.

Shared PostgreSQL Connection

We maintain a team PostgreSQL instance for development. Use read-only/restricted credentials unless you are assigned DDL tasks.

  1. Open your DB client (pgAdmin/Azure Data Studio/VS Code SQL extension).
  2. Create a new connection using the parameters below.
Connection Template (replace placeholders):
  • Host: <PG_HOST>
  • Port: 5432
  • Database: revnova_dev
  • SSL: require if mandated by host
  • User: <PG_USER_RO> (read-only)
  • Password: <PG_PASSWORD_RO>

Tip: If your VS Code has an existing saved connection, copy the parameters from the Database pane or your secrets manager. Do not commit secrets.

Local Services (Optional)

  • Run PostgreSQL/Redis locally using Docker for offline work.
# Example Docker (optional)
# docker run -p 5432:5432 -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=revnova_dev postgres:16
# docker run -p 6379:6379 redis:7-alpine

Use environment variables for local secrets: PGHOST, PGUSER, PGPASSWORD, PGDATABASE, PGPORT, REDIS_URL.

Workflow

  • Create a branch, implement, and open a PR with clear checklists.
  • Run linters and unit tests before pushing.
  • Link PRs to requirement pages when relevant.

For questions, check requirements pages or ask in the team channel. Avoid sharing credentials in chat; use the secrets vault.
