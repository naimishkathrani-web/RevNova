Quick Start (Automated Setup)

Use the automated setup script to configure your development environment in minutes:

Step 1: Clone Repository

git clone https://github.com/your-org/revnova-platform.git cd revnova-platform

Step 2: Run Setup Script

Windows (PowerShell):

.\scripts\setup-dev-env.ps1

macOS/Linux:

chmod +x scripts/setup-dev-env.sh ./scripts/setup-dev-env.sh

Step 3: Configure Environment Variables

# Copy environment template cp .env.example .env # Edit .env with your credentials # - SALESFORCE_ORG_URL # - SALESFORCE_USERNAME # - DATABASE_URL # - OPENAI_API_KEY (for AI features)

Step 4: Initialize Database

npm run db:setup # Creates database, runs migrations, seeds test data

Step 5: Authenticate with Salesforce

sf org login web --alias revnova-dev # Opens browser for OAuth flow sf org display --target-org revnova-dev # Verify connection

Step 6: Start Development Servers

npm run dev # Starts all services (frontend, backend, workers)
‚úÖ Setup Complete! Access the application at http://localhost:3000. Default credentials: admin@revnova.local / DevPassword123!

Manual Setup (Step-by-Step)

For troubleshooting or custom configurations, follow these detailed manual setup instructions:

1. Frontend Setup (React/TypeScript)

# Navigate to frontend directory cd frontend # Install dependencies npm install # Build static assets npm run build # Start development server with hot reload npm run dev # Runs on http://localhost:3000

2. Backend API Setup (Node.js/Express)

# Navigate to backend directory cd backend # Install dependencies npm install # Run database migrations npm run migrate:latest # Seed development data npm run db:seed # Start API server npm run dev # Runs on http://localhost:4000

3. Python Services Setup (AI/ML & Data Processing)

# Navigate to Python services directory cd services/python # Create virtual environment python -m venv venv # Activate virtual environment # Windows: venv\Scripts\activate # macOS/Linux: source venv/bin/activate # Install dependencies pip install -r requirements.txt # Run AI service python -m uvicorn ai_service.main:app --reload --port 5000 # Runs on http://localhost:5000

4. Database Setup (PostgreSQL)

# Create database createdb revnova_dev # Restore database schema from dump (if available) psql revnova_dev < db/schema.sql # OR run migrations to create schema npm run migrate:latest # Verify database connection psql revnova_dev -c "SELECT version();"

5. Salesforce Metadata Deployment

# Deploy custom objects and fields sf project deploy start --source-dir force-app/main/default --target-org revnova-dev # Deploy permission sets sf project deploy start --source-dir force-app/permsets --target-org revnova-dev # Assign permission set to your user sf org assign permset --name RevNova_Admin --target-org revnova-dev # Import sample data sf data import tree --plan data/sample-data-plan.json --target-org revnova-dev

6. Redis Cache Setup (Optional but Recommended)

# Using Docker (easiest method) docker run -d -p 6379:6379 --name revnova-redis redis:7-alpine # Verify Redis connection docker exec -it revnova-redis redis-cli ping # Should return "PONG"

7. Message Queue Setup (RabbitMQ for async jobs)

# Using Docker docker run -d -p 5672:5672 -p 15672:15672 --name revnova-rabbitmq rabbitmq:3-management # Access management UI: http://localhost:15672 # Default credentials: guest / guest

Troubleshooting

Common Issues & Solutions

‚ö†Ô∏è Issue: ECONNREFUSED when connecting to PostgreSQL

Solution:

  • Verify PostgreSQL is running: pg_isready
  • Check connection string in .env file
  • Ensure firewall allows port 5432
‚ö†Ô∏è Issue: Salesforce CLI authentication fails

Solution:

  • Clear cached auth: sf org logout --all
  • Use device flow: sf org login web --alias revnova-dev
  • Check Salesforce My Domain is enabled and accessible
‚ö†Ô∏è Issue: Python packages installation fails

Solution:

  • Upgrade pip: python -m pip install --upgrade pip
  • Install build tools (Windows): Install Visual Studio Build Tools
  • Use pre-built wheels: pip install --only-binary :all: -r requirements.txt
‚ö†Ô∏è Issue: Port already in use (3000, 4000, 5000)

Solution:

  • Find process using port: netstat -ano | findstr :3000 (Windows) or lsof -i :3000 (macOS/Linux)
  • Kill process or change port in .env file
‚ö†Ô∏è Issue: Docker container won't start

Solution:

  • Check Docker Desktop is running
  • Clear Docker cache: docker system prune -a
  • Restart Docker service

Development Best Practices

Code Quality

  • Run linter before committing: npm run lint
  • Write unit tests for new features (target: 80% coverage)
  • Use TypeScript strict mode (no implicit any)
  • Follow naming conventions: camelCase (JS), snake_case (Python), PascalCase (classes)

Git Workflow

  • Create feature branches from develop: git checkout -b feature/ABC-123-feature-name
  • Write descriptive commit messages: feat(migration): add CPQ data validation
  • Rebase before merging: git pull --rebase origin develop
  • Squash commits for clean history before PR

Testing Strategy

  • Unit tests: Test individual functions/components in isolation
  • Integration tests: Test API endpoints with database interactions
  • E2E tests: Cypress for critical user flows
  • Run tests before pushing: npm run test

Performance

  • Use database indexes for frequently queried fields
  • Implement caching for expensive operations (Redis)
  • Lazy load heavy components in frontend
  • Monitor bundle size: npm run build:analyze
