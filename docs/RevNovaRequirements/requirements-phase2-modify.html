Functional AI-Powered Modification Suggestions

Requirement IDDescriptionAcceptance Criteria
FR-MOD-002.1 Change Request Input • Multi-line text area for natural language requests
• Show character count (min 20 chars, max 2000 chars)
• Voice input option (speech-to-text using Web Speech API)
• Sample prompts button (show 5 example modification requests)
FR-MOD-002.2 Change Detection & Categorization • AI categorizes changes as: MODIFY, ADD, DELETE, RENAME
• Display each change type with distinct icon (✏️ Modify, ➕ Add, ❌ Delete, 🔄 Rename)
• Show before/after values for MODIFY changes
• Indicate data model impact (e.g., "This adds 2 new custom fields")
FR-MOD-002.3 AI Recommendations • Suggest related changes user may have missed (e.g., "Add corresponding discount rule?")
• Warn about potential conflicts (e.g., "This may break existing quote templates")
• Recommend best practices (e.g., "Consider tiered pricing instead of flat rate")
• Show confidence score for each suggestion (High/Medium/Low)
FR-MOD-002.4 Impact Preview • Show how many quotes, orders, contracts will be affected
• Highlight breaking changes (e.g., "Removing this feature will affect 15 active quotes")
• Estimate implementation effort (Low/Medium/High complexity)
• Display downstream system impacts (e.g., "Update CRM integration required")

Technical AI Modification Engine - Contextual Prompting

// System Prompt with Current Product Context
const MODIFICATION_PROMPT = `
You are modifying an existing Salesforce CPQ product.

Current Product Configuration:
${JSON.stringify(currentProduct, null, 2)}

User's Change Request:
"${userInput}"

Your task:
1. Identify all fields that need to change (MODIFY, ADD, DELETE, RENAME)
2. Generate new SKUs/features if requested
3. Suggest related changes user may have missed
4. Warn about potential breaking changes

Output JSON:
{
  "changes": [
    {
      "type": "MODIFY" | "ADD" | "DELETE" | "RENAME",
      "field": "field_name",
      "currentValue": any,
      "proposedValue": any,
      "confidence": 0.0-1.0,
      "reason": "why this change is needed"
    }
  ],
  "recommendations": [
    { "text": "...", "priority": "high" | "medium" | "low" }
  ],
  "warnings": [
    { "text": "...", "severity": "critical" | "warning" | "info" }
  ]
}
`;

const response = await openai.chat.completions.create({
  model: "gpt-4-turbo-preview",
  messages: [
    { role: "system", content: MODIFICATION_PROMPT },
    { role: "user", content: userInput }
  ],
  response_format: { type: "json_object" },
  temperature: 0.2 // Low temp for precise modifications
});

Technical Database Schema - Change Tracking

CREATE TABLE product_modification_history (
    modification_id UUID PRIMARY KEY,
    product_id UUID NOT NULL,
    user_id UUID NOT NULL,
    change_request TEXT NOT NULL, -- Natural language input
    ai_suggestions JSONB NOT NULL, -- Full AI response
    applied_changes JSONB, -- Subset of suggestions that were applied
    status VARCHAR(50) NOT NULL, -- 'draft', 'approved', 'rejected', 'applied'
    created_at TIMESTAMP DEFAULT NOW(),
    applied_at TIMESTAMP
);

CREATE INDEX idx_product_history ON product_modification_history(product_id, created_at DESC);
