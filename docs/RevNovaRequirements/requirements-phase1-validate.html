Functional Overview

Purpose: Validate data quality before migration using multi-stage validation (STG1‚ÜíSTG2) with categorized errors and drill-down capabilities.

STG1 (Raw)Source data extraction validationRecord count, required fields, data types, foreign keys
STG2 (Transformed)Post-transformation validationBusiness rules, field length, format, referential integrity
TargetPre-load target validationDuplicates, unique constraints, Salesforce limits (field length, etc.)

Technical Validation Rules Engine

CREATE TABLE validation_rules (
    rule_id UUID PRIMARY KEY,
    rule_name VARCHAR(255) NOT NULL,
    rule_type VARCHAR(50) NOT NULL, -- required_field, data_type, length, format, business_rule
    severity VARCHAR(20) NOT NULL, -- critical, warning, info
    error_message TEXT NOT NULL,
    sql_check TEXT NOT NULL, -- SQL query to detect violations
    auto_fix_available BOOLEAN DEFAULT false,
    auto_fix_logic TEXT
);

CREATE TABLE validation_results (
    result_id UUID PRIMARY KEY,
    migration_id UUID NOT NULL,
    validation_stage VARCHAR(10) NOT NULL, -- STG1, STG2, TARGET
    rule_id UUID NOT NULL,
    affected_record_count INT NOT NULL,
    error_details JSONB, -- Array of {record_id, field, value, error}
    validated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Example validation query
SELECT COUNT(*) as error_count
FROM stg2_product
WHERE required_field__c IS NULL; -- Detect null required fields
