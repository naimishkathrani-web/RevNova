High Level Functional Design

Functional overview of RevNova Platform across all 5 implementation phases

Phase 1: SFDC CPQ → Revenue Cloud Migration (MVP) - Due November 2025

Objective: Complete end-to-end migration platform covering all 5 workstreams (WS1-WS5) with AI-powered automation, multi-stage validation, and rollback capabilities

Core Functional Capabilities (All Workstreams)

  • WS1 Product Catalog: Product2, Pricebook2, PricebookEntry migration with AI field mapping
  • WS2 Quotes & Pricing: SBQQ__Quote__c, QuoteLine, Discount Schedules with pricing logic
  • WS3 Contracts & Assets: Subscription → Asset conversion, contract lifecycle management
  • WS4 Billing & Invoicing: Invoice generation, payment tracking, billing automation
  • WS5 Advanced Features: Product hierarchies, bundles, EPC integration

7-Page Migration Wizard

PagePurposeKey Features
1. ConnectEstablish source/target connectionsOAuth 2.0, username/password auth, encrypted credential storage
2. AnalyzeSchema discovery & readinessAuto-detect objects/fields, custom field analysis, readiness scoring
3. MapField mapping editorAI suggestions (≥85% accuracy), manual overrides, value mapping
4. TransformData transformation rulesNormalize, calculate, concatenate, split, default value transforms
5. ValidatePre-migration checksSTG1 → STG2 validation, error reporting, drill-down capability
6. ExecuteMigration executionBatch processing (200-10K), parallel workers (1-16), real-time monitoring
7. TestPost-migration verificationRecord count match, data integrity sampling, discrepancy analysis

Complete Field Mapping Coverage (70+ Fields)

  • WS1: Product2 (20+ fields), Pricebook2 (5 fields), PricebookEntry (7 fields), Account (15 fields)
  • WS2: SBQQ__Quote__c (17+ fields), SBQQ__QuoteLine__c (8 fields), SBQQ__DiscountSchedule__c (4 fields)
  • WS3: SBQQ__Subscription__c → Asset (12 fields), Contract (10 fields)
  • WS4: Invoice (14 fields), InvoiceLine (8 fields), BillingAccount (6 fields)
  • WS5: Product hierarchy (4 fields), ProductChildItem (5 fields), vlocity_cmt__PricingElement__c (6 fields)

Technical Architecture Highlights

  • Hybrid Staging Model: Vanilla tables for standard fields + EAV tables for custom fields
  • Multi-Stage Validation: STG1 (raw) → STG2 (transformed) → Target (validated)
  • AI Field Mapping: Machine learning engine with ≥85% accuracy, confidence scoring
  • Batch Processing: Configurable batch sizes, parallel workers, auto-retry on failure
  • Rollback Capability: Track inserted record IDs, one-click rollback with dependency handling
  • Real-Time Monitoring: Progress dashboard, ETA calculation, worker status
  • Audit Logging: Immutable audit trail (7 years retention), compliance-ready

Acceptance Criteria (MVP)

  • ✓ User can connect to both CPQ and RCA orgs (OAuth/username-password)
  • ✓ System auto-detects all objects (Product2, Quote, Subscription, Invoice, etc.)
  • ✓ AI suggests field mappings with ≥85% accuracy across all workstreams
  • ✓ User can manually override AI suggestions and save custom mappings
  • ✓ System validates required fields, data types, foreign keys before migration
  • ✓ Migration executes with configurable batch size (200-10K) and parallel workers (1-16)
  • ✓ Real-time dashboard displays progress, success/error counts, ETA
  • ✓ User can pause migration and resume without data loss
  • ✓ Rollback deletes all migrated records and resets project state
  • ✓ Post-migration test verifies record count match and data integrity ≥95%

Phase 2: AI Product Configuration Platform

Objective: Provide intelligent product design, configuration, and planning tools with AI-powered impact analysis and budget calculation

Core Functional Capabilities

  • Generate with AI: AI-powered product design from natural language requirements
    • Input business requirements as text (e.g., "Create a subscription product for SaaS with 3 tiers")
    • AI generates complete product specification: fields, pricing, bundles, options
    • Review and modify AI-generated product configuration
  • Modify with AI: AI-assisted product modifications
    • Upload existing product configuration
    • Request changes via natural language (e.g., "Add volume discounting")
    • AI suggests modifications with before/after comparison
  • Adopt AI Suggested Product Design: Review and approve AI recommendations
    • Side-by-side comparison of current vs. AI-suggested design
    • Field-level diff highlighting changes
    • One-click adopt or selective field adoption

User Workflows

  • Workflow 1: Product Manager describes new product → AI generates design → Review/modify → Approve → Export to Salesforce
  • Workflow 2: Upload existing product → Request AI modifications → Review changes → Adopt selected fields → Update Salesforce
  • Workflow 3: Generate product → Create marketing plan → Create selling plan → Calculate budget → Present to stakeholders

Phase 3: Multi-CPQ Platform Support

Objective: Expand source system support beyond SFDC CPQ to other CPQ platforms migrating to SFDC Revenue Cloud

Supported Source CPQ Systems

  • Oracle CPQ Cloud → SFDC Revenue Cloud
  • Apttus/Conga CPQ → SFDC Revenue Cloud
  • SAP CPQ → SFDC Revenue Cloud
  • Salesforce Native CPQ (legacy) → SFDC Revenue Cloud
  • Custom CPQ Systems → SFDC Revenue Cloud (via CSV import)

Core Capabilities

  • Universal connector framework for multi-system API integration
  • Cross-platform field mapping engine with system-specific transforms
  • Data model normalization (CPQ-agnostic staging layer)
  • System-specific business rule libraries (Oracle vs. SAP vs. Apttus logic)

Phase 4: CRM + CPQ Integration

Objective: Support CRM+CPQ combinations migrating to Salesforce RLM (Revenue Lifecycle Management) + Revenue Cloud

Supported CRM + CPQ Combinations

  • Microsoft Dynamics CRM + CPQ → SFDC RLM + Revenue Cloud
  • SugarCRM + Custom CPQ → SFDC RLM + Revenue Cloud
  • HubSpot + CPQ Integration → SFDC RLM + Revenue Cloud
  • Zoho CRM + CPQ → SFDC RLM + Revenue Cloud

Core Capabilities

  • CRM data migration (Accounts, Contacts, Opportunities, Activities)
  • Account hierarchy preservation and mapping
  • Relationship mapping (Account-Contact, Opportunity-Quote relationships)
  • CPQ + CRM data correlation and integrity validation

Phase 5: ERP System Integration

Objective: Enterprise Resource Planning systems to Salesforce RLM + Revenue Cloud for complete business transformation

Supported ERP Systems

  • SAP ERP → SFDC RLM + Revenue Cloud
  • Oracle ERP Cloud → SFDC RLM + Revenue Cloud
  • Microsoft Dynamics 365 Finance → SFDC RLM + Revenue Cloud
  • NetSuite → SFDC RLM + Revenue Cloud
  • Sage Intacct → SFDC RLM + Revenue Cloud

Core Capabilities

  • Financial data migration (GL accounts, chart of accounts, journal entries)
  • Inventory management sync (stock levels, warehouses, SKUs)
  • Order management integration (purchase orders, sales orders, fulfillment)
  • Procurement and vendor management
  • Real-time ERP ↔ Salesforce sync for ongoing operations
