The 7-page wizard processes these object types using the same unified architecture:

  • WS1 - Product Catalog: Product2 (20+ fields), Pricebook2 (5 fields), PricebookEntry (7 fields)
  • WS2 - Quotes & Pricing: SBQQ__Quote__c (17+ fields), QuoteLine (8 fields), DiscountSchedule (4 fields)
  • WS3 - Contracts & Assets: Subscription â†’ Asset (12 fields), Contract (10 fields)
  • WS4 - Billing & Invoicing: Invoice (14 fields), InvoiceLine (8 fields), BillingAccount (6 fields)
  • WS5 - Advanced Features: Product hierarchy (4 fields), ProductChildItem (5 fields), PricingElement (6 fields)

Note: Each object type flows through the same 7-page wizard â€” not separate wizards per workstream

Cross-Workstream Platform Capabilities

  • Unified Data Model: Generic object schema that can represent any source/target object type
  • Polymorphic Transformation Engine: Same transformation rules apply to Products, Quotes, Contracts, etc.
  • Generic Validation Framework: Validation rules are object-agnostic (required fields, data types, referential integrity)
  • Hybrid Staging Model: Vanilla tables for standard fields + EAV tables for custom fields (works for ALL objects)
  • Orchestration Engine: Manages migration workflows for any object type with dependency resolution

Acceptance Criteria for Phase 1 MVP

  2. Successfully migrate â‰¥1 sample object from each workstream (WS1-WS5) using the same platform
  3. AI field mapping achieves â‰¥85% accuracy on standard fields
  4. Validation engine catches â‰¥95% of data quality issues pre-migration
  5. Batch processing handles 10,000+ records with <5% failure rate
  6. Post-migration testing confirms 100% record count accuracy
  7. Platform architecture is extensible (can add new object types without code changes)
  8. Rollback functionality tested and working for all object types
  9. Real-time monitoring dashboard shows batch progress across all workers
  10. Complete documentation of object-oriented architecture for Phase 3-5 reuse
