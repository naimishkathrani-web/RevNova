Testing & Parity Matrix — CPQ → Revenue Cloud

This page defines comprehensive test scenarios to validate migration accuracy, pricing parity, and functional equivalence between CPQ and Revenue Cloud Advanced (RCA). Each test compares source and target systems across structure, pricing, lifecycle, and data quality dimensions.

WS1 — Product Catalog Testing

Scenario ID Scenario Name Source → Target Data Set Type Primary Objects Test Dimension What to Compare CPQ Measurement RCA Measurement Tolerance Test Method APIs / Tools Priority Status Notes
PC-01 Simple standalone product parity CPQ → RCA Master Data Product2 → vlocity_cmt__Product__c Structure # of active products by family in scope COUNT(Product2 WHERE IsActive=TRUE) COUNT(EPC Products WHERE Active__c=TRUE AND mapped_from_CPQ=TRUE) 0 variance SOQL compare or exported CSV /services/data/vXX.X/query on Product2 and vlocity_cmt__Product__c High Not Started Run once after initial product migration
PC-02 Bundle structure parity CPQ → RCA Master Data SBQQ__ProductOption__c → vlocity_cmt__ProductChildItem__c Structure # of child options per parent bundle AVG/COUNT(SBQQ__ProductOption__c GROUP BY SBQQ__Product__c) AVG/COUNT(ProductChildItem__c GROUP BY ParentProductId__c) 0 variance in # of options per bundle SOQL or API export of options CPQ ProductModel API, EPC product/child-item query High Not Started Spot-check some complex bundles manually in UI
PC-03 Config attribute parity CPQ → RCA Master Data SBQQ__ConfigurationAttribute__c → ConfigAttribute / AttributeValue Structure # of attributes and allowed values per product COUNT(ConfigAttributes) and COUNT(AllowedValues) COUNT(RCA attributes and values mapped) 0 variance Data export to CSV and comparison CPQ object exports + RCA ConfigAttribute objects Medium Not Started Ensure all required attributes exist with correct picklist values
